# M28 Coupon Redemption and Order Discounts Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (\`- [ ]\`) syntax for tracking.

**Goal:** 구매자가 단일 상품 주문에 발급 쿠폰 하나를 선택해 예약·결제 승인·할인 스냅샷을 안전하게 처리하게 만듭니다.

**Architecture:** \`MemberCoupon\`은 발급·사용 상태를, 새 \`CouponReservation\`은 주문 진행 중 예약 이력과 만료를 책임집니다. 주문 생성은 쿠폰을 잠근 뒤 가격 스냅샷과 예약을 저장하고, 결제 승인 또는 0원 내부 승인은 예약을 소비하면서 쿠폰을 \`USED\`로 바꿉니다.

**Tech Stack:** Java 21, Spring Boot, Spring Data JPA, PostgreSQL/Flyway, React/TypeScript, Vite, JUnit 5, MockMvc, Testcontainers.

    id bigint generated by default as identity primary key,
    member_coupon_id bigint not null references member_coupons(id),
    order_id bigint not null unique references orders(id),
    status varchar(20) not null,
    reserved_at timestamp with time zone not null,
    expires_at timestamp with time zone not null,
    consumed_at timestamp with time zone,
    released_at timestamp with time zone,
    constraint chk_coupon_reservations_status check (status in ('RESERVED','CONSUMED','RELEASED','EXPIRED'))
);

    on coupon_reservations(member_coupon_id) where status = 'RESERVED';
alter table orders add column member_coupon_id bigint references member_coupons(id);
alter table orders add column coupon_discount_amount bigint not null default 0;

\`\`\`java
public CouponDiscountQuote quote(MemberCoupon coupon, Product product, PromotionPrice promotion, Instant now) {
    requireIssuedAndValid(coupon, now);
    requireTargetMatches(coupon, product.getId());
    requirePromotionStackable(coupon, promotion.promotionId());
    long base = promotion.effectivePrice();
    requireMinimumPurchase(coupon, base);
    long discount = coupon.getDiscountType() == FIXED_AMOUNT
            ? Math.min(coupon.getDiscountValue(), base)
            : Math.min(percentageFloor(base, coupon.getDiscountValue()), maxDiscount(coupon));
    return new CouponDiscountQuote(coupon.getId(), discount, Math.max(0L, base - discount));
}
\`\`\`

정률 계산은 \`(base / 100) * rate + ((base % 100) * rate) / 100\`으로 버림과 오버플로 회피를 보장합니다. \`validUntil\`, \`ISSUED\`, 대상 상품, 최소 금액, 프로모션과의 \`stackable\`만 검사하며 이후 캠페인 상태는 검사하지 않습니다. 지갑 상태와 JPQL 필터에서도 캠페인 중지·종료를 \`UNAVAILABLE\`로 만드는 조건을 제거합니다. 적용 가능 목록은 활성 예약이 없는 쿠폰만 할인액과 예상 최종가를 반환합니다.
